# 代理路由器

根据任务描述自动选择最合适的子代理。

## 可用代理

| 代理 | 角色 | 触发条件 |
|------|------|----------|
| **architect** | 架构师 | 系统设计、技术选型、模块划分 |
| **coder** | 编码者 | 写代码、实现功能、修复 bug |
| **reviewer** | 审查员 | 代码审查、安全审计、质量检查 |
| **tester** | 测试员 | 编写测试、执行测试、覆盖率分析 |
| **devops** | 运维 | CI/CD、部署、监控、基础设施 |
| **docs** | 文档 | API 文档、README、变更日志 |
| **debugger** | 调试专家 | 排查问题、性能分析、日志解读 |

## 路由逻辑

1. 分析用户任务描述
2. 提取关键词和意图
3. 匹配代理能力
4. 如有多个匹配，选择最专业的
5. 将任务连同上下文转发给选定代理
6. 收集代理输出，格式化返回

## 使用方式

直接描述任务，路由器自动分派：
- "帮我设计一个用户认证模块" → architect
- "这个函数有 bug" → debugger
- "给这个 API 写文档" → docs
- "部署到 staging" → devops
