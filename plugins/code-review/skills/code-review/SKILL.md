# 多 Agent 并行代码审查

启动多个子代理从不同维度并行审查代码，覆盖面更广、深度更深。

## 审查维度

### 1. Bug 检测 Agent
- 空指针和异常处理
- 边界条件覆盖
- 并发安全
- 资源泄漏（文件句柄、连接池）
- 类型安全

### 2. 安全审计 Agent
- SQL 注入、XSS、CSRF
- 敏感信息硬编码
- 权限校验缺失
- 依赖漏洞
- 输入验证

### 3. 性能 Agent
- N+1 查询
- 不必要的循环嵌套
- 大对象未释放
- 缓存策略合理性
- 数据库索引缺失

### 4. 规范 Agent
- 命名规范一致性
- 代码重复（DRY）
- 函数职责单一
- 注释完整性
- 错误码规范

## 使用流程

1. 指定待审查的代码范围（文件、目录或 diff）
2. 并行启动 4 个维度的审查 Agent
3. 汇总各 Agent 的发现
4. 按严重程度排序输出报告

## 报告格式

```
## 代码审查报告

### 严重 (必须修复)
- [文件:行号] 问题描述 + 修复建议

### 警告 (建议修复)
- [文件:行号] 问题描述 + 修复建议

### 建议 (可选优化)
- [文件:行号] 优化建议
```

## 注意事项

- 审查范围明确指定，避免全量扫描
- 优先关注变更代码，非变更代码仅标记异常
- 安全问题一律标为严重级别
