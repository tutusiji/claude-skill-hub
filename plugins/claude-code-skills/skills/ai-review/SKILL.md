# 多模型 AI 代码审查

用多个 AI 模型交叉审查代码，取交集提高置信度。

## 工作流程

1. 获取变更 diff (git diff main...HEAD)
2. 分别用不同模型审查（Claude / GPT / Gemini）
3. 合并发现，标注哪些模型一致指出
4. 按置信度排序：多模型一致 > 单模型发现

## 审查维度
- **安全性** — 注入、越权、信息泄露
- **正确性** — 逻辑错误、边界条件、空值
- **性能** — N+1 查询、不必要计算、内存泄漏
- **可维护性** — 命名、复杂度、重复代码

## 输出格式

```
## 多模型代码审查报告

### 高置信（多模型一致）
- [file:line] ⚠️ <发现> — Claude ✓ GPT ✓ Gemini ✓

### 中置信（两模型一致）
- [file:line] <发现> — Claude ✓ GPT ✓ Gemini ✗

### 低置信（单模型发现，需人工判断）
- [file:line] <发现> — 仅 GPT 发现

### 汇总
N 个高置信问题，M 个中置信，K 个低置信
建议: <修复高置信问题后可合并>
```
