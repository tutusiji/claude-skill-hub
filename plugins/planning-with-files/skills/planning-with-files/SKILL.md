# 持久化项目规划

将项目规划和任务进度持久化到文件系统，实现跨会话上下文保留。

## 规划文件结构

```
project-root/
  .plan/
    overview.md          # 项目概述和目标
    tasks/
      001-auth-module.md  # 认证模块
      002-api-design.md   # API 设计
      003-database.md     # 数据库设计
    progress.md           # 进度追踪
    decisions.md          # 技术决策记录
```

## 任务文件模板

```markdown
# 任务：[任务名称]

## 状态
[ ] 未开始 / [~] 进行中 / [x] 已完成

## 背景
[为什么需要这个任务]

## 目标
- [具体目标 1]
- [具体目标 2]

## 步骤
1. [ ] 步骤一
2. [ ] 步骤二
3. [ ] 步骤三

## 备注
[技术细节、依赖关系、风险点]
```

## 使用流程

1. **初始化规划** — 在项目根目录创建 `.plan/` 结构
2. **拆解任务** — 将需求分解为可独立完成的任务文件
3. **记录进度** — 每次工作前读取 `progress.md`，工作后更新状态
4. **跨会话恢复** — 新会话开始时读取规划文件，快速恢复上下文

## 设计原则

- 每个任务文件独立可读，不依赖其他文件
- 状态用 checkbox 标记，一眼可见进度
- 技术决策单独记录，方便回溯
