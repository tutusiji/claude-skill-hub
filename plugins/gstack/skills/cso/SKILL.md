# 安全审计 (CSO)

使用 OWASP Top 10 和 STRIDE 方法论执行全面安全审计。

## 流程

1. 确定范围：全代码库或最近变更（询问用户）
2. 执行 OWASP Top 10 检查清单
3. 执行 STRIDE 威胁建模
4. 生成发现报告

## OWASP Top 10 检查清单

1. **注入** — SQL、NoSQL、OS 命令、LDAP 注入
2. **认证失效** — 弱会话、缺失 MFA、撞库
3. **敏感数据泄露** — 明文存储、弱加密、TLS 问题
4. **XXE** — XML 外部实体处理
5. **访问控制失效** — 缺失鉴权检查、IDOR、路径遍历
6. **安全配置错误** — 默认凭证、详细错误信息、开放调试
7. **XSS** — 反射型、存储型、DOM 型
8. **不安全反序列化** — 对象注入、通过反序列化 RCE
9. **已知漏洞组件** — 有 CVE 的过期依赖
10. **日志不足** — 缺少安全事件审计日志

## STRIDE 威胁建模

| 威胁 | 检查内容 |
|------|----------|
| **伪装 (S)** | 能否冒充其他用户？ |
| **篡改 (T)** | 数据在传输或存储中能否被修改？ |
| **抵赖 (R)** | 操作能否被否认（无审计日志）？ |
| **信息泄露 (I)** | 数据能否泄露给未授权方？ |
| **拒绝服务 (D)** | 服务能否被瘫痪？ |
| **权限提升 (E)** | 用户能否获取未授权的访问级别？ |

## 输出格式

```
## 安全审计报告

### 范围
<代码库或相对 main 的 diff>

### 严重（立即修复）
- [文件:行号] 描述 + 修复方案

### 高危（下个版本前修复）
- [文件:行号] 描述 + 修复方案

### 中危（下个迭代修复）
- [文件:行号] 描述 + 修复方案

### 低危 / 信息
- [文件:行号] 描述

### STRIDE 汇总
- 伪装: <发现>
- 篡改: <发现>
- 抵赖: <发现>
- 信息泄露: <发现>
- 拒绝服务: <发现>
- 权限提升: <发现>
```
