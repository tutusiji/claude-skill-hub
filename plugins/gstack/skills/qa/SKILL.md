# QA 测试

打开真实浏览器，访问目标 URL，执行功能测试。

## 流程

1. 从用户获取 URL（或使用最近部署的 staging URL）
2. 打开浏览器访问该 URL
3. 执行以下测试清单
4. 带截图输出结果

## 测试清单

### 加载
- 页面无错误加载（检查控制台）
- 资源无 404（CSS、JS、图片）
- 页面在合理时间内渲染（< 3 秒）

### 功能
- 所有交互元素可用（按钮、表单、链接）
- 表单校验正确触发
- 页面间导航正常
- 会话/认证状态按预期保持

### 视觉
- 桌面视口无布局破坏
- 无重叠元素
- 文字可读（对比度、字号）
- 图片加载且尺寸正确

### 边界情况
- 空状态（无数据、无结果）
- 错误状态（网络失败、无效输入）
- 长文本不破坏布局

## 输出格式

```
## QA 报告：<url>

### 通过
- [测试项] 描述

### 失败
- [测试项] 描述 + 截图

### 阻塞
- [测试项] 无法运行，因为...

### 结论
<可以发布 / 先修复 N 个问题>
```
