# 代码审查技能

按照公司编码规范审查代码变更。

## 工作流程

1. 读取 diff 或待审查文件
2. 检查安全问题（硬编码密钥、注入漏洞、不安全反序列化）
3. 检查代码质量（命名、复杂度、死代码、缺失错误处理）
4. 检查最佳实践（输入校验、日志、测试覆盖）
5. 输出结构化审查报告，按严重级别分类：阻断 / 警告 / 建议

## 输出格式

```
## 代码审查报告

### 阻断问题
- [文件:行号] 描述

### 警告
- [文件:行号] 描述

### 建议
- [文件:行号] 描述

### 汇总
N 个阻断，M 个警告，K 个建议。
```
