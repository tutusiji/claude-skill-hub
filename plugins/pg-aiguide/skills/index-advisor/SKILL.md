# 索引顾问

智能推荐索引策略，平衡查询性能与写入开销。

## 分析项

### 1. 缺失索引检测
- 分析常用查询的 WHERE/JOIN/ORDER BY 字段
- 对比现有索引，找出未覆盖的查询模式
- 推荐复合索引（考虑字段顺序）

### 2. 冗余索引检测
- 找出从未被使用的索引（浪费写入性能）
- 找出重叠索引（(a,b) 和 (a) 同时存在）
- 建议删除无用索引

### 3. 索引选择性分析
- 低选择性索引（如 boolean 字段）→ 不建议单独索引
- 高选择性字段优先放复合索引前面

## 输出格式

```
## 索引分析报告

### 建议创建
- CREATE INDEX idx_xxx ON table(col1, col2);
  原因: 查询 "WHERE col1=? AND col2=?" 出现 N 次/天
  预期: 查询提速 X 倍

### 建议删除
- DROP INDEX idx_yyy;
  原因: 过去 30 天零使用，每次写入额外开销 Xms

### 复合索引优化
- 现有: (a) → 建议改为: (a, b)
  原因: 覆盖更多查询模式
```
