# SQL 优化

分析并优化 PostgreSQL 查询性能。

## 工作流程

1. 获取慢查询 SQL 和执行计划 (EXPLAIN ANALYZE)
2. 识别性能瓶颈：全表扫描、嵌套循环、排序溢出
3. 提供优化方案：
   - 索引建议（哪些字段需要索引）
   - 查询重写（子查询改 JOIN、避免 SELECT *）
   - 分区建议（大表分区策略）
4. 估计优化效果

## 常见优化模式

- **缺索引**：WHERE/JOIN 字段无索引 → 建议创建
- **N+1 查询**：循环中执行查询 → 改为批量查询
- **全表扫描**：Seq Scan → 检查 WHERE 条件是否命中索引
- **排序溢出**：Sort 节点 cost 高 → 增加工作内存或加排序索引
- **过度 JOIN**：多表 JOIN 性能差 → 考虑物化视图或反范式
