# 影响分析

分析代码变更的影响范围，防止意外破坏。

## 工作流程

1. 确定变更目标：哪个函数/类/变量被修改
2. 构建依赖图：谁直接调用、谁间接调用
3. 标注影响层级：直接影响（1 跳）、间接影响（2-3 跳）
4. 识别高风险路径：公共 API、测试覆盖薄弱的路径
5. 生成影响报告

## 输出格式

```
## 影响分析：修改 <symbol>

### 直接影响 (N 处)
- [file:line] <调用方式>

### 间接影响 (M 处)
- [file:line] → [file:line] <链路>

### 高风险路径
- [file:line] ⚠️ 公共 API，可能影响外部调用方
- [file:line] ⚠️ 无测试覆盖

### 建议
- 先补充测试: <未覆盖的路径>
- 考虑向后兼容: <公共接口>
```
