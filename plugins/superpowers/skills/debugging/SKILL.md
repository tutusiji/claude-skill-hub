# 系统化调试

有纪律地查找和修复 bug。永远不猜 — 永远验证。

## 流程

### 1. 复现
- 建立可靠的最小复现
- 不能复现就不能修复
- 尽可能将复现写成测试用例

### 2. 隔离
- 二分查找：注释掉一半代码，bug 还在吗？
- 精简输入：去掉所有非必要的部分
- 定位行为偏离期望的确切代码行

### 3. 假设
- 形成具体可证伪的假设："bug 在 X 因为 Y"
- 写下什么证据能确认或否定这个假设

### 4. 验证
- 添加日志或测试来检验假设
- 确认 → 修复根因
- 否定 → 形成新假设，不修补症状

### 5. 修复
- 修复根因，不是症状
- 写回归测试：修复前失败，修复后通过
- 验证修复没有破坏其他测试

### 6. 确认
- 运行完整测试套件
- 检查其他地方是否有类似 bug（相同模式，不同位置）
- 在提交信息中记录发生了什么以及为什么

## 规则
- 不修复无法复现的 bug
- 不写没有回归测试的修复
- 不在根因还在时修补症状
- 如果调试超过 30 分钟，退一步从头重新读代码
- "以前是好的" → `git bisect` 是你的朋友
