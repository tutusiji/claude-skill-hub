# 头脑风暴

在确定方案之前先探索问题空间。

## 流程

1. **理解** — 用自己的话复述问题，和用户确认。
2. **发散** — 生成至少 3 种可能方案，先不评判。
   - 最明显的方案是什么？
   - 最小化的方案是什么？
   - 最大胆的方案是什么？
3. **评估** — 对每个方案：
   - 它解决了什么问题？
   - 它在哪里行不通？
   - 成本是多少（时间、复杂度、依赖）？
4. **收敛** — 选择一个方案，解释为什么。标注不做什么以及为什么。
5. **确认** — 用 2-3 句话总结计划。问："这是你想要的吗？"

## 规则
- 头脑风暴期间不写代码
- 总是提供至少 3 种方案，即使一种明显最优
- 尊重 YAGNI — 最小化方案通常是对的
- 如果用户已有清晰计划，直接跳到规格编写
- 挑战假设："我们真的需要数据库吗？"
